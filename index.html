Fallback-Reihenfolge

1) worldcup26.ir (ohne Key) → 2) KickoffAPI (mit Key) → 3) WorldCupAPI (mit Key) → 4) eingebetteter Snapshot.

Was Multi-API-Fallback bedeutet

Wenn ein Feed ausfällt, nutzt V4 automatisch den nächsten Feed. Falls nur ein Live-Overlay verfügbar ist, werden Snapshot-Daten als Basis erhalten und um Live-Scores ergänzt.

🛠 Anleitung: Teams, Spiele und Provider im Dashboard anpassen

1) Teams / Ranking anpassen → suche nach const RANKINGS = { ... }.

const RANKINGS = {
  "Argentina": 1,
  "Germany": 10,
  "Mexico": 14,
  "New Team": 42
};

2) Flaggen anpassen → suche nach const TEAM_FLAGS = { ... }.

const TEAM_FLAGS = {
  "Argentina": "🇦🇷",
  "Germany": "🇩🇪",
  "New Team": "🏳️"
};

3) Gruppen ändern → suche nach const GROUPS = { ... }.

const GROUPS = {
  A: ["Mexico", "South Africa", "Korea Republic", "Czechia"]
};

4) Snapshot-Ergebnisse ändern → suche nach SNAPSHOT.completedMatches und bearbeite Spiele.

{
  id: 1,
  kickoff: "2026-06-11T21:00:00+02:00",
  group: "A",
  home: "Mexico",
  away: "South Africa",
  homeScore: 2,
  awayScore: 0,
  venue: "Mexico City Stadium",
  stage: "group"
}

5) Kommende Spiele ändern → suche nach SNAPSHOT.upcomingMatches.

{
  id: 41,
  stage: "Gruppenphase",
  group: "J",
  home: "Argentina",
  away: "Austria",
  kickoff: "2026-06-22T19:00:00+02:00",
  venue: "AT&T Stadium, Dallas"
}

6) Bracket ändern → suche nach SNAPSHOT.bracket.

bracket: {
  r32: [
    {
      id: 73,
      code: "M73",
      date: "2026-06-28T21:00:00+02:00",
      labelHome: "A2",
      labelAway: "B2",
      venue: "Los Angeles Stadium"
    }
  ]
}

7) Provider-Reihenfolge ändern → suche nach const PROVIDERS = [ ... ]. Dort kannst du weitere Quellen ergänzen oder die Reihenfolge umstellen.

Falls keine Live-Quelle erfolgreich antwortet, bleibt das Dashboard mit dem eingebetteten Snapshot vollständig funktionsfähig.
